BNF — Basis HTML

Een grammatica die beschrijft welke HTML-documenten geldig zijn

non-terminal <...>
terminal "..."
lege string ε
recursieve regel (nesting)
Document
<document> ::=
<html-el>
<html-el> ::=
"<html>" <head-el> <body-el> "</html>"
<head-el> ::=
"<head>" <title-el> "</head>"
<title-el> ::=
"<title>" <tekst> "</title>"
<body-el> ::=
"<body>" <inhoud> "</body>"
Inhoud & nesting
<inhoud> ::=
<element> <inhoud>
ε
recursief: inhoud = nul of meer elementen achter elkaar
<element> ::=
<div-el>
<p-el>
<h-el>
<lijst>
<tekst>
<div-el> ::=
"<div>" <inhoud> "</div>"
inhoud verwijst terug → nesting werkt voor elke diepte
<p-el> ::=
"<p>" <inline> "</p>"
<h-el> ::=
"<h1>" <inline> "</h1>"
"<h2>" <inline> "</h2>"
"<h3>" <inline> "</h3>"
Lijsten
<lijst> ::=
"<ul>" <items> "</ul>"
"<ol>" <items> "</ol>"
<items> ::=
"<li>" <inline> "</li>" <items>
ε
recursief: nul of meer li-elementen
Inline inhoud
<inline> ::=
<tekst> <inline>
"<a>" <tekst> "</a>" <inline>
"<strong>" <tekst> "</strong>" <inline>
ε
<tekst> ::=
… willekeurige tekenreeks (geen tags) …

Afleiding — hoe de grammatica een geldig document genereert

1<document>
2<html> <head-el> <body-el> </html>
3<html><head><title-el></head> <body-el> </html>
4<html><head><title><tekst></title></head> <body-el> </html>
5<html><head><title>"Mijn pagina"</title></head> <body-el> </html>
6…<body><inhoud></body></html>
7…<body><element> <inhoud> </body>
8…<body><div><inhoud></div> <inhoud> </body>
9…<body><div><p>Hallo</p></div></body></html>
Cruciaal op stap 8: <div> gebruikt <inhoud> — dezelfde non-terminal als het document zelf. Daardoor kan een div een andere div bevatten, die weer een div bevat, enzovoort — voor elke diepte. Geen enkele FSA kan dit, omdat een FSA niet bijhoudt hoeveel tags er nog openstaan.

FSA — kan dit niet

  • Tellen hoeveel tags openstaan
  • Controleren of sluitag bij openingstag hoort
  • Nesting op willekeurige diepte
  • Onthouden wat er eerder stond

BNF — kan dit wel

  • Recursieve regels → nesting ingebakken
  • Structuur afdwingen (</div> hoort bij <div>)
  • Elke gewenste diepte beschrijven
  • Taal van HTML, code, JSON precies vastleggen
